Safety Rules — Read Before You Type
AMHS is a critical aviation system — messages affect flight safety
Never run commands that change/delete files on the live server without approval
Always BACKUP config files before editing: cp file file.bak_$(date +%Y%m%d)
Confirm your hostname before acting: run hostname first
Never restart network service when connected remotely
All READ commands in this training are safe: cat, ls, df, ps, grep, tail
Document every action: time, command used, output, and who approved it

cd /var/log/amhs

# Go back up

Linux for AMHS | RHEL 5.59
💾
Always Backup Before Editing
Golden Rule: Copy → Edit → Verify. Never edit original files directly.
TERMINAL
# Step 1: Go to AMHS config directory
cd /etc/amhs
# Step 2: Backup with timestamp (ALWAYS do this first)
cp -p amhs.conf amhs.conf.bak_$(date +%Y%m%d_%H%M)
# Step 3: Verify backup was created
ls -la amhs.conf*
# Step 4: Only NOW edit the config
# (after getting approval from senior admin)

03

Reading Logs

tail, less, grep — find errors and trace AMHS messages

📜
Why Logs Matter for AMHS
✈️

Every AFTN/AMHS message is logged

Incoming and outgoing aviation messages are recorded as text entries

🔍

Errors appear in logs before failures

Catching 'connection refused' or 'disk full' early prevents outages

⏱️

Timestamps help reconstruct events

When did the message arrive? When did it fail? Logs tell you exactly

📋

Logs are required for incident reports

Authorities require documented evidence of what happened and when

Linux for AMHS | RHEL 5.512
👁️
Log Viewing Commands
⚠️Never use cat on large log files — it will freeze your terminal
less /var/log/messages
less — Page-by-page reader. Arrow keys to scroll. Press q to quit. Safe — cannot modify file.
tail -n 100 /var/log/messages
tail -n — Last N lines. Shows the most recent 100 lines. Adjust N as needed.
tail -f /var/log/messages
tail -f — Live follow mode. Streams new lines in real-time. Press Ctrl+C to stop.
Linux for AMHS | RHEL 5.513
🔍
grep — Search Inside Logs

grep finds specific keywords inside any text file — essential for AMHS troubleshooting

grep -i "error" /var/log/messages
← Find all errors (case-insensitive)
grep -i "fail|reject|timeout" /var/log/amhs.log
← Search for multiple keywords at once
grep -r "connection refused" /var/log/
← Search recursively through all logs
zgrep ERROR messages-20260101.gz
← Search inside compressed (rotated) logs
grep -i error /var/log/messages | tail -20
← Last 20 errors — combine with tail

Process & Service Management

Check if AMHS is running and manage system services

⚙️
Process Monitoring

Check if AMHS is running

ps -ef | grep amhs
Lists all processes. Pipe to grep to filter AMHS. If no output → AMHS is NOT running.

Real-time CPU & RAM usage

top
Shows processes by CPU usage live. Press q to exit. Look for AMHS process and its % usage.
TERMINAL
# Example: Is AMHS running?
$ ps -ef | grep amhs
amhs  1234  1  0  09:15 ?
   /usr/amhs/bin/amhsd
amhs  1235  1234  0  09:15 ?
   /usr/amhs/bin/amhsmgr
# 2 lines = AMHS is running ✓
# 0 lines = AMHS is NOT running ✗
Linux for AMHS | RHEL 5.516
🔧
Service Management — RHEL 5.5 Style
RHEL 5.5 uses SysV init — use service and chkconfig (NOT systemctl)
SAFE
service amhs status
Check if AMHS service is running
SAFE
service network status
Check network service status
SAFE
chkconfig --list | grep amhs
See which runlevels AMHS starts at
CAUTION
service amhs restart
Restart AMHS — requires senior admin approval
DANGER
service network restart
Will drop your remote connection if done via SSH

Network for AMHS

Check IP addresses, test connectivity, verify open ports

🌐
Network Commands
🔌

Show IP addresses

ifconfig -a

See all network interfaces and their IP addresses on this server

📡

Test host reachability

ping 192.168.1.50

Send test packets to remote station. Ctrl+C to stop. No reply = not reachable.

🗺️

View routing table

route -n

Shows how network traffic is directed. Numeric format (-n) avoids DNS lookups.

👂

See listening ports

netstat -an | grep LISTEN

Shows which ports the server is waiting for connections on

🔗

Test port connectivity

telnet host 102

Checks if a specific port is open. Port 102 is used by X.400/AMHS.

🛣️

Trace network path

traceroute host

Shows each hop to destination — useful to find where connection fails

Disk, Memory & Permissions

Monitor resources and understand file security

💿
Disk & Memory Monitoring
⚠️If /var or /var/log is 100% full, AMHS will immediately stop processing messages!
df -h
Disk usage — all filesystems. Check /var and / columns. Alert if 'Use%' is above 80%.
du -sh /var/log/*
Size of log directories. Find which log folder is consuming the most disk space.
free -m
RAM usage in megabytes. Check 'free' column. Low free RAM may slow AMHS processing.
vmstat 1 5
System load snapshot (5 samples). Shows CPU, memory and I/O every 1 second. Use when system feels slow.
Linux for AMHS | RHEL 5.521
🔐
Users & File Permissions
id amhs
Show user ID and group memberships
groups amhs
List all groups the user belongs to
chmod 640 file
Set file permissions (owner:rw, group:r, others:none)
chown amhs:amhs f
Change file owner and group
su -
Switch to root — use only when necessary
READING PERMISSIONS
-rw-r-----

-   = regular file
rw- = owner: read+write
r-- = group: read only
--- = others: no access

Backup & Troubleshooting

Protect configs and respond to AMHS incidents

📦
Backup & File Transfer Commands
tar -cvf amhs_config.tar /etc/amhs
Create archive. Bundles entire /etc/amhs folder into one file
gzip amhs_config.tar
Compress archive. Reduces size. Creates amhs_config.tar.gz
tar -xvf amhs_config.tar.gz
Extract archive. Restore from backup when needed
scp file user@host:/path
Copy file to remote server. Securely transfer backup to another machine via SSH
md5sum amhs_config.tar.gz
Verify file integrity. Generate checksum to confirm file was not corrupted during transfer

🚨
AMHS Incident Response Checklist

Follow this order every time AMHS has a problem

1
Confirm server identity:hostname date who
You are on the right machine, at the right time
2
Check disk space:df -h
Is /var or /var/log full? Full disk = messages stop
3
Is AMHS running?:ps -ef | grep amhs
If no output, AMHS process is down
4
Check service status:service amhs status
Confirms whether service is active or stopped
5
Read last 100 log lines:tail -n 100 /var/log/messages
Look for recent errors around the time of the issue
6
Search for keywords:grep -i "error|fail|reject" log
Narrow down the cause
7
Check network connectivity:ifconfig ping host netstat -an
Can the server reach the remote station?

⛔
Commands to Avoid as a Beginner
HIGH
rm -rf /path
Deletes files permanently with no recovery. Always verify path with ls first.
HIGH
chmod -R 777 .
Removes all security from files. Anyone can read/modify AMHS configs.
MED
kill -9 <PID>
Forces process to stop without cleanup. May corrupt AMHS message queue.
HIGH
service network restart
Drops your remote SSH session. Only do this at the physical console.
HIGH
Editing config without backup
If something breaks, you cannot restore the original settings.
