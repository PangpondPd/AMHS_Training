Lab 0: Setup — Create Your Practice Environment
Run these commands first — sets up all files used in later labs
SETUP

Run this one-time setup in your Ubuntu VM terminal:

# Create practice directory tree
mkdir -p ~/amhs-practice/logs ~/amhs-practice/backup ~/amhs-practice/config
# Create a fake AMHS config file
echo 'hostname=AMHS-SERVER-01' > ~/amhs-practice/config/amhs.conf
echo 'port=102'              >> ~/amhs-practice/config/amhs.conf
echo 'protocol=X400'        >> ~/amhs-practice/config/amhs.conf
# Create fake log files
for i in 1 2 3; do
  echo "[INFO]  AMHS service started" > ~/amhs-practice/logs/amhs.log.$i
  echo "[INFO]  Connection from 192.168.1.$i" >> ~/amhs-practice/logs/amhs.log.$i
  echo "[ERROR] Timeout on port 102" >> ~/amhs-practice/logs/amhs.log.$i
  echo "[INFO]  Retry successful"    >> ~/amhs-practice/logs/amhs.log.$i
done
# Create a text file for practice
echo 'AFTN EGGGZQZX' > ~/amhs-practice/message.txt
echo 'FF VHHHYNYX'   >> ~/amhs-practice/message.txt
echo 'END OF MESSAGE' >> ~/amhs-practice/message.txt
echo 'Setup complete! Run: ls ~/amhs-practice/'

KEY THINGS TO VERIFY

✔ ls -lh shows permissions, owner, size, date
✔ ls -lt shows newest files at the top
✔ cp leaves original, mv does not
✔ After rm, the file is completely gone
✔ Tab key auto-completes 'rep' to 'reports/'

Common mistakes to watch for:
✘ Forgetting the trailing / in cp dest
✘ Using rm -r on a directory accidentally
✘ Confusing cp (copy) with mv (move/rename)

chmod numeric quick reference:
  777 = rwxrwxrwx (everyone — AVOID)
  755 = rwxr-xr-x (scripts/directories)
  644 = rw-r--r--  (regular files)
  640 = rw-r-----  (sensitive config files)
  600 = rw-------  (private keys, passwords)
chown command (change ownership):
  chown user:group filename
  chown amhs:amhs /etc/amhs/config
  (requires root/sudo on real server)
